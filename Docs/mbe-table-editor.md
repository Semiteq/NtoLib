# MbeTableEditor — Редактор рецептов MBE

> **Кратко:** Функциональный блок MasterSCADA 3.12 для подготовки технологических рецептов без обмена с
ПЛК: создание и редактирование строк, импорт/экспорт CSV, работа с буфером обмена и
статический расчёт времени рецепта. Запускается без подключённого контроллера. В палитре
MasterSCADA отображается как отдельный блок «Редактор рецептов MBE».

## 1. Интерфейс

У блока нет входов и выходов обмена с ПЛК, мониторинга исполнения и Modbus-адресации.
Единственные входы блока — динамические входы имён оборудования, создаваемые автоматически по
конфигурации (см. раздел «Конфигурация»). Они служат источником значений для выпадающих
списков и проверки имён при импорте CSV, а не каналами обмена с контроллером.

### Входы (имена оборудования)

Состав групп и число входов в каждой задаются файлом `PinGroupDefs.yaml` (поле `pin_quantity`),
а не фиксированы в коде. Значения в таблице ниже — пример для шаблона конфигурации MBE
(например, 27 у `HeaterNames` — это `pin_quantity` для нагревателей в этом шаблоне):

| Группа | Тип | Кол-во входов | Описание |
|--------|-----|--------------|----------|
| ShutterNames | string | 16 | Имена заслонок |
| HeaterNames | string | 27 | Имена нагревателей |
| NitrogenSourcesNames | string | 16 | Имена источников азота |
| ChamberHeaterNames | string | 16 | Имена нагревателей камеры |

Для других конфигураций (например MOCVD) состав групп будет иным — он целиком задаётся
`PinGroupDefs.yaml`.

### Выходы

Выходов нет. Результат работы блока — отредактированный рецепт, сохраняемый в файл CSV.

## 2. Визуальные свойства (окно настроек)

| Категория | Свойство | По умолчанию | Описание |
|-----------|----------|--------------|----------|
| Вычисления | Эпсилон | `0.0001` | Допуск сравнения чисел с плавающей точкой при расчёте времени |
| Конфигурация | Путь к конфигурационному каталогу | `C:\DISTR\Config\MBE` | Каталог с YAML-файлами конфигурации таблицы. Изменения вступают в силу при следующей загрузке блока |
| Логирование | Записывать лог в файл | выкл. | Включает запись операций в лог-файл |
| Логирование | Путь к каталогу логов | `C:\DISTR\Logs\MbeTableEditor` | Каталог для лог-файлов. Допускаются переменные окружения |

Если запись лога включена, операции пишутся в файл в указанном каталоге логов. Ротация — по
размеру файла (5 МБ); хранится до 5 последних файлов.

Помимо перечисленных, блок предоставляет тот же набор свойств визуального оформления, что и
MbeTable: высота строки; цвета фона блока, статусной строки, таблицы и кнопок; шрифты и цвета
заголовка таблицы, обычной строки, текущей строки, пройденной строки и заблокированной ячейки.
Свойства текущей и пройденной строки задают цвета подсветки исполнения — в редакторе она не
активна (нет обмена с ПЛК). Свойство «Цвет фона» задаёт фон всего блока; значение
`Color.Transparent` игнорируется (см. известную проблему #80).

Свойства Modbus (IP, порт, UnitId, базовые адреса, размеры областей, тайм-ауты и т.п.) у
этого блока отсутствуют.

## 3. Конфигурация

Блок использует те же четыре YAML-файла, что и полный блок MbeTable: `PropertyDefs.yaml`,
`PinGroupDefs.yaml`, `ColumnDefs.yaml`, `ActionsDefs.yaml`. Каталог задаётся свойством «Путь к
конфигурационному каталогу» (по умолчанию `C:\DISTR\Config\MBE`). Чтобы редактор и полный
блок работали с одинаковыми рецептами, укажите им один и тот же каталог.

Если путь пуст или не задан, редактор останавливается с явной ошибкой — в отличие от
полного блока MbeTable, который тихо подставляет каталог установки MasterSCADA. Это
намеренное расхождение: содержимое сериализованного свойства непредсказуемо между
обновлениями палитры (поле могло отсутствовать в старой версии или иметь другое
значение по умолчанию), и для редактора молчаливая загрузка не того набора YAML опаснее остановки.
У полного блока тихая подстановка каталога сохранена ради совместимости с проектами, созданными до
появления свойства.

Секция `PlcMapping` в `ColumnDefs.yaml` допускается, но редактором игнорируется.

Формат конфигурационных файлов и формат CSV полностью совпадают с полным блоком —
см. [документацию MbeTable](mbe-table.md) (разделы про конфигурацию и импорт/экспорт CSV).

## 4. Работа с редактором

Работа с редактором аналогична работе с таблицей рецептов в полном блоке MbeTable, за исключением отсутствия обмена с ПЛК. Работа с буфером обмена так же поддерживается.

Результат операций, сообщения об ошибках и расчётное время рецепта (общее и по шагам)
отображаются в статусной строке в левом нижнем углу блока. Расчёт времени статический —
выполняется по самому рецепту.

Удалена кнопка «Записать в контроллер», отправка и чтение рецепта по Modbus TCP, 
а также подсветки текущего шага и обратного отсчёта времени во время 
исполнения. Смотри также [документацию MbeTable](mbe-table.md).

## 5. Отличия от блока MbeTable

| Возможность | MbeTable | MbeTableEditor |
|-------------|----------|----------------|
| Редактирование рецепта (строки, ячейки) | Да | Да |
| Импорт/экспорт CSV | Да | Да |
| Буфер обмена (копировать/вставить) | Да | Да |
| Статический расчёт времени рецепта | Да | Да |
| Динамические входы имён оборудования | Да | Да |
| Отправка рецепта в ПЛК (Modbus TCP) | Да | Нет |
| Чтение рецепта из ПЛК | Да | Нет |
| Мониторинг исполнения (подсветка шага, отсчёт времени) | Да | Нет |
| Кнопка «Записать в контроллер» | Есть | Нет |
| Свойства и входы/выходы Modbus/мониторинга | Есть | Нет |

Оба блока — самостоятельные элементы палитры с разными идентификаторами; они регистрируются
вместе и не конфликтуют. Редактор не обращается к контроллеру, поэтому добавляется и работает
на машине без настроенного ПЛК.
